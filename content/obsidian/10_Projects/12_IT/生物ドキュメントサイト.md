# 生物ドキュメントサイト

## 1. 背景・目的

生態学について学習した内容（概念、数式、モデル、用語、参考文献メモ等）を体系的に整理し、後から参照・拡張できる「まとめサイト」を構築する。

特に以下を重視する：

- **Markdown/MDXで編集しやすい**
- **インタラクティブグラフを表示可能**（計算はユーザー端末＝ブラウザで実行）
- **コメント可能**
- **軽量**（必要なページだけJSを読み込み、静的配信を基本とする）

---

## 2. ゴール / 非ゴール

### 2.1 ゴール（達成したいこと）
- MD/MDXファイルの編集でページを追加・更新できる
- 記事内にインタラクティブなグラフ（スライダー等）を埋め込める
- 各記事にコメント欄を設置できる（外部サービス利用可）
- 静的ホスティングで運用でき、表示が軽快
- Gitで履歴管理できる（GitHub管理は“必須ではない”が、利用する）

### 2.2 非ゴール（今回やらない）
- サイト側でログイン機能、ユーザー管理、ロール管理を実装しない
- サイト側でコメントDB/APIを自前実装しない（giscus等に委譲）
- サーバサイドで数値計算（R/Python等）を実行しない
- 編集をブラウザ完結のCMSに寄せない（ローカル編集を基本とする）

---

## 3. 想定ユーザー・利用シーン

### 3.1 主ユーザー
- サイト管理者（筆者）：生態学の学習内容を整理して公開/非公開で蓄積する

### 3.2 利用シーン
- 研究・勉強中に概念をまとめて記事化
- 図やモデル挙動を、パラメータ操作で直感的に確認
- 読者（または未来の自分）がコメントで補足・議論

---

## 4. システム概要

- **サイト種別**：静的サイト（Static Site）
- **コア技術**：Astro + MDX
- **インタラクティブ**：ページ内コンポーネントとしてJS実行（必要ページのみ）
- **コメント**：giscus（GitHub Discussionsをコメント保存先として利用）
- **ホスティング**：静的ホスティング（例：Cloudflare Pages / Netlify / GitHub Pages いずれでも可）

---

## 5. 技術スタック要件

### 5.1 フロントエンド/ビルド
- Astro（最新版追従、LTS概念がないため**定期アップデート前提**）
- MDX（Markdownにコンポーネント埋め込み）
- Node.js：LTS（推奨：最新LTSを採用、CIでも同一バージョン）

### 5.2 記事管理
- Gitリポジトリで管理
- 編集はローカル（Obsidian / VS Code 等）を基本

### 5.3 コメント
- giscus を採用
  - 目的：静的サイトに**コメント投稿・表示**を付与する（自前DB不要）
  - 保存先：GitHub Discussions（リポジトリ単位）
  - 認証：GitHubログイン（スパム抑制に寄与）

### 5.4 グラフ/可視化（軽量優先）
- 既定：軽量な可視化ライブラリ（例：ECharts / uPlot / lightweight-charts 等）
- 追加：表現力が必要な場合のみ Plotly/D3 を採用（**ページ単位で遅延読み込み**）
- 方針：**必要なページだけJSを読み込む**（全ページ共通バンドルに巨大ライブラリを入れない）

---

## 6. 機能要件（Functional Requirements）

### 6.1 コンテンツ（記事）管理
**FR-01** 記事は `.md` / `.mdx` で作成できる  
**FR-02** 記事にはFrontmatter（メタ情報）を持てる  
**FR-03** 記事一覧（最新順）を生成できる  
**FR-04** タグ/カテゴリで記事を分類できる  
**FR-05** ドラフト（下書き）をビルドから除外できる

推奨Frontmatter（例）：
- `title`（必須）
- `description`（任意）
- `date`（必須）
- `updated`（任意）
- `tags`（任意）
- `draft`（任意, boolean）
- `math`（任意, boolean：数式表示の有無）
- `toc`（任意, boolean：目次表示の有無）

### 6.2 インタラクティブグラフ
**FR-10** MDX内でグラフコンポーネントを埋め込める  
**FR-11** グラフに必要な計算はブラウザ（クライアント）で行う  
**FR-12** データは静的ファイルとして同梱できる（JSON/CSV 等）  
**FR-13** 重い計算がある場合、UIが固まらないよう分離できる（Web Worker等）

設計方針：
- グラフは `src/components/plots/*` に集約
- 大きい依存（Plotly等）は `dynamic import` で遅延ロード
- データは `public/data/*` に置き、`fetch('/data/xxx.json')` で読む

### 6.3 コメント
**FR-20** 記事ページにコメント欄を表示できる  
**FR-21** コメントの投稿/表示は外部サービスを利用し、サイト側にDBを持たない  
**FR-22** 記事とコメントスレッドの紐付け方式を選べる（URL/タイトル/ID等）

giscusの推奨設定：
- `mapping`: pathname（URL単位でスレッド作成）
- `reactions`: 有効（任意）
- `theme`: サイトテーマと連動（ライト/ダーク切替対応）

### 6.4 ナビゲーション/情報設計
**FR-30** グローバルナビ（トップ、タグ、カテゴリ、検索、About等）  
**FR-31** パンくず or 階層（必要なら）  
**FR-32** 関連記事表示（同タグ等）

### 6.5 検索（任意だが便利）
**FR-40** 静的サイト向け検索を提供できる（例：Pagefind等）  
- 目的：サーバ不要で全文検索を実現
- 優先度：Should（後回し可）

---

## 7. 非機能要件（Non-Functional Requirements）

### 7.1 パフォーマンス / 軽量性
**NFR-01** 初期表示の軽快さを重視（不要なJSを配らない）  
**NFR-02** グラフ系ライブラリはページ単位ロード  
**NFR-03** 画像は最適化（適切なサイズ、遅延読み込み、WebP等）

目標（目安）：
- Lighthouse（モバイル）で Performance 80+ を目標（達成困難なページは例外定義）

### 7.2 可用性
- 静的ホスティング前提（CDN配信）
- 依存サービス（giscus/GitHub）が落ちた場合でも本文閲覧は可能

### 7.3 保守性
- コンポーネント（グラフ）を再利用可能な形で整理
- 依存ライブラリ更新を容易に（大物依存を局所化）
- コード整形：Prettier（推奨）
- 型：TypeScriptは任意（導入する場合はコンポーネントから段階的に）

### 7.4 セキュリティ
- 外部埋め込み（giscus等）を想定し、CSP（Content-Security-Policy）の検討
- Dependabot等で脆弱性アップデートを追跡（推奨）

### 7.5 SEO / メタ
- `sitemap.xml` 生成
- OGP/Twitterカード
- canonical URL設定
- RSS（任意）

---

## 8. 画面・ページ要件（最低限）

### 8.1 ページ種別
- トップ（最新記事一覧）
- 記事ページ（MD/MDX）
- タグ一覧 / タグ別一覧
- カテゴリ一覧 / カテゴリ別一覧（必要なら）
- About（サイト説明）
- 検索（導入する場合）

### 8.2 記事ページの標準構成
- タイトル、日付、タグ
- 目次（任意）
- 本文（MD/MDX）
- コメント欄（giscus、任意でON/OFF可能）

## 10. 開発・運用要件

### 10.1 開発フロー
- ローカル編集 → Git commit → push → CIでビルド → 静的ホスティングへデプロイ

### 10.2 CI/CD（推奨）
- GitHub Actionsで以下を実施
  - `pnpm/npm install`
  - `astro build`
  - （任意）リンクチェック、Lint
  - プレビュー環境（PRごと）※ホスティング側機能でも可

### 10.3 ホスティング
- いずれか（どれでも成立）
  - Cloudflare Pages
  - Netlify
  - GitHub Pages
- 重要：**静的配信ができること**、**ビルドコマンドが設定できること**

---

## 11. 受け入れ基準（Acceptance Criteria）

- `.mdx` を追加すると新しい記事ページが生成される
- 記事内にグラフコンポーネントを埋め込め、操作に応じて表示が変化する
- グラフの計算はブラウザで実行され、サーバサイドは不要
- 記事ページにgiscusコメント欄が表示され、投稿・閲覧ができる
- グラフのない通常記事は軽快に表示される（不要JSを極力含まない）

---

## 12. 優先度（MoSCoW）

### Must（必須）
- MD/MDX編集でページ生成
- インタラクティブグラフ（クライアント計算）
- コメント（giscus）
- 軽量化（ページ単位JS）
- タグ/カテゴリ整理
- 静的全文検索（Pagefind等）

### Should（できれば）
- RSS / sitemap / OGP

### Could（余裕があれば）
- 数式表示の最適化（KaTeX等）
- 関連記事の高度化
- ダークモード完全対応（giscusテーマ連動含む）

---

## 13. 未決定事項（現時点の推奨デフォルト）
- 可視化ライブラリ：まずは軽量寄り（ECharts等）で開始し、必要ページのみPlotly追加
- 検索：後からPagefindを追加（初期はタグ/カテゴリで代替）
- ホスティング：Cloudflare Pagesを推奨（速度・運用が楽）。ただしどれでも可
