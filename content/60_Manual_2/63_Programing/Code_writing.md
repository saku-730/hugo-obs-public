クソコードを書かないために。



## 変数名

キャメルケース(パスカルケース)かスネークケース

それぞれの意味は
[「キャメルケース」と「パスカルケース」の違い｜「分かりそう」で「分からない」でも「分かった」気になれるIT用語辞典](https://wa3.i-3-i.info/diff697moji.html)
[「キャメルケース」と「スネークケース」の違い｜「分かりそう」で「分からない」でも「分かった」気になれるIT用語辞典](https://wa3.i-3-i.info/diff71moji.html)

- キャメルケース：testCode
- パスカルケース：TestCode
- スネークケース：test_code

基本はその言語の慣習に従おう。どっちでもいい場合にはスネークケースでいく。そっちのほうが見やすい。

## 保守

あとから見直して何をやっているのかわかり、改修しやすいコードをかけ。コメントで説明するのもいいが、そもそものアルゴリズムや書き方をわかりやすくすべし。

小さいことは良いことである。(small is beautiful)

## ヘッダコメント

ソースコードの一番上(ヘッダー)に色々作成者とかライセンスの情報を書いておく。

```
########
#Script Name: 
#Description: 
#Date: 
#Version: 1.0
#Dependencies:
#- 
#
#Usage:
#
########
```
